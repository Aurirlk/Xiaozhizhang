# NeuVox AI CRM 系统说明文档

## 一、系统概述

AI CRM（客户关系管理）系统是 NeuVox 的核心模块之一，用于自动提取和管理用户画像信息。

### 核心功能

1. **用户画像提取** - 从对话中自动提取用户姓名、职业、偏好等信息
2. **意图识别** - 识别用户意图（天气查询、新闻获取、搜索、知识库、闲聊）
3. **用户行为分析** - 记录和分析用户交互行为
4. **跨会话记忆** - 用户画像持久化存储，支持跨会话断点续传

│  分支1:        │   │  分支2:        │   │  分支3:        │
│  法律查询      │   │  天气/搜索等   │   │  闲聊对话      │

│               │   │    Calling    │   │               │

### 2.2 三分支详细说明

#### 分支1: 法律查询 → 知识库 RAG

| 步骤 | 说明 |
|------|------|
| 意图识别 | `IntentType.KNOWLEDGE` |
| 触发条件 | 包含"知识库"、"专业"、"法律"、"规定"等关键词 |
| 处理方式 | 调用 `query_knowledge` 工具 |
| 数据源 | ChromaDB 向量检索 + SQLite 全文检索 |
| 回复生成 | LLM 基于检索结果生成自然语言回复 |

#### 分支2: 天气/搜索等 → Function Calling

| 意图 | 工具 | 数据源 |
|------|------|--------|
| `WEATHER` | `get_weather` | 和风天气 API |
| `NEWS` | `get_trending_news` | RSS 新闻源 |
| `SEARCH` | `web_search` | Bing 搜索 |
| `TIME` | `get_time` | Python 硬编码 |

#### 分支3: 闲聊对话 → 直接 LLM

| 步骤 | 说明 |
|------|------|
| 意图识别 | `IntentType.CHAT` |
| 触发条件 | 其他所有情况 |
| 处理方式 | 直接调用 LLM 生成回复 |
| 特点 | 无工具调用，纯对话 |

```
用户对话

    ▼
┌─────────────────────────────────────────────────────────┐
│                    IntentRouter                         │

└───────────────────────────┬─────────────────────────────┘
                            │
                            ▼

┌─────────────────────────────────────────────────────────┐
│                  SQLite 数据库                           │

└─────────────────────────────────────────────────────────┘
```

## 三、CRM 工作流程

### 3.1 用户画像提取流程

```
用户对话
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  1. 调用 LLM 分析对话内容                                │
│     - 提取姓名、性别、职业、城市等                        │
│     - 提取偏好信息（爱好、饮食、音乐）                     │
│     - 提取意向和预算范围                                  │
└───────────────────────────┬─────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  2. JSON 解析                                          │
│     - 清理 LLM 响应（移除 markdown 标记）                 │
│     - 解析为结构化数据                                   │
└───────────────────────────┬─────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  3. 数据合并                                           │
│     - 获取现有用户画像                                   │
│     - 合并新数据（保留旧数据，更新新数据）                  │
│     - 偏好列表去重合并                                   │
│     - 标签去重合并                                       │
└───────────────────────────┬─────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│  4. 存入 SQLite                                        │
│     - 更新 user_profiles 表                             │
│     - 更新 interactions 表的 crm_extracted_data          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 意图识别流程

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  1. 关键词匹配（快速）                                   │
│     - 检查天气、新闻、搜索等关键词                        │
│     - 如果匹配，返回意图                                  │
└───────────────────────────┬─────────────────────────────┘
                            │
                   匹配成功?
                   ┌────┴────┐
                   │         │
                   ▼         ▼
                  是         否
                   │         │
                   ▼         ▼
              返回意图   ┌────────────────────────────────┐
                         │  2. LLM 意图识别（兜底）        │
                         │     - 调用大模型分析意图         │
                         │     - 返回意图类型和实体         │
                         └────────────────────────────────┘
```

## 四、数据存储

### 4.1 SQLite 数据库

- **数据库文件**: `data/neuvox.db`
- **表结构**: users, interactions, user_profiles
- **特点**: 轻量级、零配置、适合单机部署

### 4.2 ChromaDB 向量数据库

- **数据库目录**: `data/chroma_db/`
- **用途**: 存储用户画像的向量表示
- **特点**: 支持语义检索，用于 RAG

## 七、配置说明

### 7.1 CRM 配置（.config.yaml）

```yaml
CRM:
  CRMAnalyzer:
    enabled: true
    async_analysis: true
    extraction_fields:
      - name
      - gender
      - age_range
      - occupation
      - city
      - preferences
      - intent
      - budget_range
      - tags
```

### 7.2 意图识别配置

```yaml
Intent:
  IntentRouter:
    type: keyword_and_llm
    enabled: true
    confidence_threshold: 0.8
    intents:
      weather:
        keywords: [天气, 气温, 温度, 下雨]
        tool: get_weather
      news:
        keywords: [新闻, 热点, 大瓜]
        tool: get_trending_news
      search:
        keywords: [搜索, 查一下, 是什么]
        tool: web_search
      knowledge:
        keywords: [知识库, 专业, 领域]
        tool: query_knowledge
      time:
        keywords: [时间, 几点, 日期, 今天]
        tool: get_time
```

## 十、性能优化

### 10.1 异步处理

- CRM 分析在后台异步执行，不阻塞主对话流程
- 使用 `asyncio.create_task()` 或 Celery 任务队列

### 10.2 缓存策略

- 用户画像缓存在 ChromaDB 中
- 支持语义检索快速查询

### 10.3 批量处理

- 支持批量导入用户数据
- 支持批量触发 CRM 分析
